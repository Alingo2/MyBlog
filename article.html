1.搭建个人博客

发布于:2019-7-27 阅读: 标签:#排坑
如果按时间记录的话，第一个要分享的当然就是搭建自己的专属博客啦~
现在流行的博客框架有不少，我选择的是hexo（配置过程十分简单，并且使用的人很多，可以排不少坑，还有多种主题可以选择）
下面我将分5步向大家介绍一下如何搭建自己的博客
1. 前置工作
2. 配置hexo
3. 修改主题
4. 上传自己的html js css文件
5. 将hexo主页直接跳转到自己的html网页上
 1. 前置工作
首先，你需要准备好1个github账号，并在电脑上安装完成 node.js
Next,在github上新建一个名为 你的用户名.github.io的仓库，比如说，如果我的github用户名是Alingo2，那么你就新建ALingo2.github.io的仓库（必须是你的用户名，其它名称无效）
 2. 配置hexo
第一步完成了~恭喜
现在，对于windows系统，打开命令提示符，键入以下指令：npm install -g hexo
(也就是利用npm帮你在全局中配置hexo)
然后，在电脑中选择一个你喜欢的位置建一个文件夹，然后cd进入该文件的目录，在终端键入指令：hexo init
(也就是hexo帮你进行初始化)
初始化完成后，键入指令：
  hexo g
(g代表generate，hexo会帮你渲染文件)
再键入:
  hexo s
(s代表server，hexo会开启本地预览服务)
复制给出的地址http://localhost:4000到浏览器,你应该会惊喜地发现有东西啦！这是hexo帮你生成的一篇HelloWorld文章
Ctrl + C 退出本地服务，接下来我们就可以把它推到远端github仓库了！ 打开你创建的文件夹，然后用文本编辑器打开站点配置文件 _config.yml，翻到最后，修改为 deploy:
type: git
repo: https://github.com/YourgithubName/YourgithubName.github.io.git
branch: master
(YourgithubName就是你的GitHub账户) 这个时候需要先安装deploy-git ，也就是部署的命令,这样你才能用命令部署到GitHub。
在cmd中键入指令: npm install hexo-deployer-git --save
然后键入：
  hexo clean
  hexo g
  hexo d
注意deploy时可能要你输入username和password。过一会儿就可以在http://yourname.github.io 这个网站看到你的博客了！！
 3.修改主题
你可能不太喜欢现在这个大地球矬矬的亚子╮(╯▽╰)╭，没关系，hexo的优点之一就是有很多别人写好的主题，只需要自己更换就好啦 第一步当然是你需要去下载你需要的主题，下边以 NexT 主题为例，
在cmd中键入指令：
  cd 博客存放的目录
  git clone https://github.com/iissnan/hexo-theme-next themes/next
克隆完成后，打开 _config.yml 文件，将主题修改为 next：
theme: next
然后 依然是那三板斧：
hexo clean
hexo g
hexo s
确认本地服务器没问题之后，退出，推到远端
  hexo d
恭喜你！大功告成啦！
 4.上传自己的html js css文件
虽然hexo模板众多，也可以修改模板的css文件，但如果你想完全自己动手又怎么办呢？
我在接触到hexo的时候就很想把自己的文件传上去，在尝试过程中还是遇到了一些坑，但终归还是实现啦(开森(*￣▽￣*)ブ)
首先我们在Hexo的source文件夹下新建一个你自己的文件夹，比如我取名为 'about'
然后，我们要知道我们打开的html网页其实是hexo帮你把你写好的markdown文章渲染出来的，那么我就需要阻止hexo渲染你自己的文件
这里介绍一种官方文档给出的方法：
使用文本编辑器打开 你的Hexo 目录下的_config.yml 文件，找到 skip_render项
修改成：
  skip_render: '你的文件夹/**'
其中，'你的文件夹'用你自己的文件夹替代，'/**代表skip跳过渲染该文件夹下所有文件及子目录'(如果有另外的需求可以去查一下hexo skip_render规则) 例如，我就填入的'about/**'
好了，现在我们就可以大胆地把js css html等等文件都放进去了~
下一步，已经有了网页了，现在我们需要将我们的网页链接到hexo生成的主页上去
找到theme目录，用文本编辑器打开你之前配置的主题文件夹中的_config.yml文件，在menu那一栏下添加：about: /about/index.html (因为我添加的文件夹名为about,我需要链接的网页为index.html)
添加完成后，老规矩，三板斧:
hexo clean
hexo g
hexo s
打开本地的localhost，你就会发现侧栏多出了about的标签，点击就能进入你自己的网页啦！
如果没有问题,再键入:
hexo d
推到远端，就能让别人也登录你的网站啦！
 5. 将hexo主页直接跳转到自己的html网页上
如果已经成功做完之前步骤的小伙伴还没有开始动手写html可以不做第5步, 可以就把hexo生成的页面当作自己站的首页，稍加修改就可以只利用它来跳转自己我网页。
但因为我自己的网站也已经写好了个人首页，于是我决定直接跳过hexo生成的首页，我这里用了个非官方做法，但好在十分简单：
既然替换或者删掉hexo原本的网页可能很麻烦，那为什么不onload页面就直接跳转呢？
进入hexo文件中的source文件夹，编辑首页的md文件 加入一行代码就行：
  <script type="text/javascript" src="/js/jump.js"> </script>
而我在js/jump.js中的代码是：
  window.location.href="about/index.html";
(其中about/index.js是我的网站文件)
注：jump.js应该放在
  主题名/source/js 路径下！
这一步完成之后，老规矩先用本地服务器验证一下(还是那三步)，你应该会惊喜地发现进入网站后应该停留了一下然后马上就跳转到了自己的网页 (虽然有点像钓鱼网站的说...不过好歹也实现了完整地搭建自己的原生博客！！！)
